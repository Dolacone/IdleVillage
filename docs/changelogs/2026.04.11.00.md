# Change Plan & Changelog: 2026.04.11.00

- Status: Draft

## 1. Objectives
- Enhance UI/UX for both the public announcement board and the /idlevillage interface.
- Simplify monster threat display and active villager status.
- Provide better clarity on building costs within the interaction menu.

## 2. Proposed Changes

### [ ] UI: Announcement & Dashboard Refinement
- Village Threat Section:
  - Format Change: Move monster details to a new line and use an emoji prefix.
  - Detail Reduction: Remove Max HP and Quality percentage from the summary line to reduce clutter.
  - New Template:
    ```
    Village Threat:
    ⚠️ {Monster_Name} (HP: {hp})
    ```
- Active Villagers Section:
  - Format Change: Simplify the action name for combat to "Attacking {Monster_Name}" without HP or Quality details.
  - Goal: Ensure the aggregated list is clean and readable at a glance.

### [ ] UI: Action Menu (/idlevillage) Enhancements
- Build Sub-menu:
  - Description Update: Append the specific resource types required for each building to the dropdown option description.
  - Example: "Food cost reduction (materials: food & wood)".
- Dashboard Alignment:
  - Ensure the /idlevillage embed description matches the new Village Threat multi-line format defined for the announcement.

## 3. Affected Files
- docs/interfaces/announcement.md: Update the unified message template and mapping rules.
- docs/interfaces/player_commands.md: Update the build sub-menu description format.
- src/core/engine.py: Update `render_announcement` to support the new multi-line threat format and simplified villager status.
- src/cogs/actions.py: Update building dropdown descriptions to include resource costs.
