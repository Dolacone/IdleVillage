# Change Plan & Changelog: 2026.04.08.03

- Status: In-Progress

## 1. Objectives
- Rebalance building XP decay to be sustainable for small villages while maintaining difficulty for large-scale progression.
- Implement a non-linear decay model that scales with both active population and current building level (XP).
- Target milestones:
  - Level 2: Easy to maintain for all village sizes.
  - Level 5: Requires consistent effort (approx. 10-15% of active population).
  - Level 9: Practically impossible due to exponential maintenance costs.
- Update Village Announcement format to align with /idlevillage UI (Resources and Buildings).
- Implement a UI-driven management interface for administrators to replace the manual /idlevillage-admin command.
- Add building upgrade announcements to celebrate village progression.

## 2. Proposed Changes

### [ ] Dynamic Exponential Decay Model
- Formula: `Decay_per_Cycle = int(Active_Players * 0.5) + int(Current_XP * (0.001 + Current_XP / 5,000,000))`
- Logic:
  - Base Population Decay: `Active_Players * 0.5` ensures that even low-XP buildings have a small maintenance cost that scales with village size.
  - Dynamic XP Decay: Base 0.1% ratio increases linearly with XP (`XP / 5M`), creating an exponential curve.
  - Implementation: Update `Engine.settle_village` in `src/core/engine.py`.

### [ ] UI Alignment: Village Announcement
- Change: Split the announcement into rich-text and code-block sections.
- Markdown Section: **Village Resources** and **Village Buildings** use emojis and bold text (outside code block).
- Code Block Section: **Active Villagers** list remains inside a code block for column alignment.
- Implementation: Update `Engine.render_announcement` in `src/core/engine.py`.

### [ ] Admin Management UI: `/idlevillage-manage`
- Command: `/idlevillage-manage` (Ephemeral, Admin Only).
- UI Flow:
  1. Main Menu: Select "Manage Resources" or "Manage Nodes".
  2. Resource Mode: Select type (Food/Wood/Stone) -> Show current amount -> Buttons (+100, +1000, -100, -1000, Set Custom Modal).
  3. Node Mode: Select node from dropdown (shows ID, Type, Stock, Quality) -> Click "Remove Node" (Red button).
- Implementation: Add `ManageView` and `ManageModal` to `src/cogs/general.py`.

### [ ] Building Upgrade Announcements
- Logic: In `Engine._settle_time_slice`, compare building level before and after XP addition.
- Action: If level increased, call `Engine.send_upgrade_announcement`.
- Implementation: Update `src/core/engine.py`.

## 4. Verification & Testing Requirements

### 4.1 Decay Simulation (Math Verification)
- Test Case A (Small Village): 5 Active Players, 3,000 XP (Lv2).
  - Expected Decay: `int(5 * 0.5) + int(3000 * (0.001 + 3000/5M))` = `2 + 3` = **5 XP/Cycle**.
  - Maintenance: Requires ~0.2 builders (5 XP / 25 XP_per_builder).
- Test Case B (Mid Village): 10 Active Players, 31,000 XP (Lv5).
  - Expected Decay: `int(10 * 0.5) + int(31000 * (0.001 + 31000/5M))` = `5 + 31 * (1 + 0.006)` = `5 + 33` = **38 XP/Cycle**.
  - Maintenance: Requires ~1.5 builders.
- Test Case C (Extreme): 100 Active Players, 511,000 XP (Lv9).
  - Expected Decay: `int(100 * 0.5) + int(511000 * (0.001 + 511000/5M))` = `50 + 511 * (1 + 0.102)` = `50 + 563` = **613 XP/Cycle**.
  - Maintenance: Requires ~24.5 builders.
