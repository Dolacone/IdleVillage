# Change Plan & Changelog: 2026.04.13.00

- Status: Draft

## 1. Objectives
- Personal Tokens: Players earn category-based tokens (Gathering, Exploring, Building, Attacking). Tokens are used for personal buffs (3x cycle), village protection (1x cycle, 50% decay reduction), or Village Commands (10 tokens).
- Village Commands: Players can spend 10 tokens to set a village-wide action. Idle players will automatically perform this action if conditions (resources/nodes) are met.
- Exploration Rebalance: Fix the discovery formula and equalize chances (25%) for all resources and threats. Implement a node capacity cap (2x storage).
- Threat & Combat Re-logic: Repurpose monsters as a persistent "Threat" node. Threats accumulate HP slowly (10% of resource yield), cause double decay when over threshold (Active Villagers * 100), and yield resources/gold per damage dealt.

## 2. Proposed Changes

### Database Updates (`src/database/schema.py`)
- [ ] Add `tokens` table: `(player_discord_id, village_id, token_type, amount)`.
- [ ] Add `player_buffs` table: `(player_discord_id, village_id, buff_type, expires_at)`.
- [ ] Update `villages` table: Add `active_command` and `protection_expires_at`.
- [ ] Repurpose `monsters` table: Remove `expires_at`, ensure `hp` is used as the threat value.

### Engine Logic (`src/core/engine.py`)
- [ ] Token Yielding: In settlement, grant 1 token (except for `idle`) per completed Action Cycle.
- [ ] Self-Buff Logic: 1 token = 3 * ACTION_CYCLE_MINUTES. Different buff types override existing ones and clear remaining time. Matching actions gain +100 total stats in the efficiency formula while the buff is active.
- [ ] Village Protection: 1 token = 1 * ACTION_CYCLE_MINUTES. Time stacks linearly. Reduces village decay by 50%.
- [ ] Village Command: If `status == 'idle'`, attempt to perform `villages.active_command`. Fallback to `idle` (food) if pre-deduction fails or nodes are empty.
- [ ] Exploration Fix: 
    - Formula: `(AGI + PER) / 2 / 100 * EXPLORING_BASE_CHANCE`.
    - Equal 25% chance for Food, Wood, Stone, or Threat.
    - Resource nodes and Threat HP capped at `2 * Storage_Capacity`.
    - Threat HP increment = `calculated_resource_yield * 0.1`.
- [ ] Decay Calculation: Double decay if `monster_hp > (active_villagers * 100)`. Apply 50% reduction if protection buff is active.
- [ ] Attack Settlement: Yield 1 Gold and 1 Random Base Resource (Food/Wood/Stone) per 1 Damage dealt. Remove monster when HP <= 0. No extra rewards.

### Commands Layer (`src/cogs/actions.py`, `src/cogs/general.py`)
- [ ] Add `/idlevillage-tokens` for all token-related actions, including status, self-buff usage, and village protection usage.
- [ ] Add `/idlevillage-village-command` for village-wide command setting, with clear UI messaging that setting the command will spend 10 tokens.
- [ ] Update dashboard: Show active Village Command, Protection status, and Threat warning (if > threshold).
