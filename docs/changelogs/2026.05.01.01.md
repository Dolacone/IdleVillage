# 2026.05.01.01 - Engine: formulas, action resolver, cycle timing

Status: Done

## Goal

Implement the pure v2 gameplay engine from `docs/engine/formula.md`, `docs/engine/action-resolver.md`, and `docs/engine/cycle-engine.md`.

## Implementation Gap

- `src/core/engine.py` still contains v1 stats, buffs, tokens, monsters, and storage behavior.
- Current cycle logic is not based on per-player `completion_time`, `last_update_time`, and `ACTION_CYCLE_MINUTES`.
- Burst, partial cycle, shortage penalty, and material drop rules do not match v2.
- Canonical stored enum values must be used for actions and materials while Discord UI displays Traditional Chinese.

## Scope

- Implement raw output:
  - `raw_output = base_output x (1 + stage_bonus + gear_bonus + facility_bonus)`
  - floor all stored calculations.
- Implement complete cycle settlement.
- Implement partial cycle settlement for action change.
- Implement burst as three independent complete-cycle settlements.
- Use canonical action stored values: `gathering`, `building`, `combat`, `research`.
- Use canonical material columns: `materials_gathering`, `materials_building`, `materials_combat`, `materials_research`.
- Store `action_target` only for `building`; map `research` to `research_lab` internally.
- For first-time action setup with `last_update_time = null`, skip partial settlement and initialize `last_update_time = now`.
- Implement shortage handling:
  - deduct each required resource down to a minimum of 0.
  - apply a single `output x0.5` penalty flag.
  - do not apply shortage penalty to stage progress.
  - do not apply shortage penalty to material drops.
- Implement material roll only for complete cycles.
- Implement cycle catch-up with `MAX_CYCLES_PER_SETTLEMENT`.

## Out of Scope

- No Discord component routing.
- No public notification sending.
- No dashboard rendering.
- No standalone manager feature work beyond the writes required by settlement orchestration.
- Engine settlement may call manager/repository functions to write resources, building XP, stage progress, material drops, and player cycle timestamps.

## Files

- `src/core/engine.py` or new focused modules under `src/core/`
- `src/database/schema.py` only if helper queries need small additions
- focused tests under `tests/`

## Verification

- Complete cycle updates `last_update_time` to the settled cycle end time.
- Complete cycle advances `completion_time` by one cycle period per settled cycle.
- Partial cycle uses proportional floor cost and output.
- First-time action setup with null `last_update_time` does not run partial settlement.
- Partial cycle does not grant materials.
- Action validation accepts only canonical stored values.
- Research action writes XP to `research_lab` without requiring `action_target`.
- Burst spends AP through player rules but does not update `last_update_time` or `completion_time`.
- Burst rolls material drop three times.
- Resource shortage applies one output penalty flag only.

## Dependencies

Requires `2026.05.01.00`.
