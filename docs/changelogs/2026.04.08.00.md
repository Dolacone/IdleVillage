# Change Plan & Changelog: 2026.04.08.00

- Status: Draft

## 1. Objectives
- **Player Readiness**: Polish `/idlevillage` UI with gaming-oriented names and structured layout.
- **Public Dashboard**: Implement Village Announcement (mono-spaced public status message).
- **Engine Refactor**: Transition from real-time hour cycles to flexible `Action Cycle Duration` units.
- **Mathematical Depth**: Gaussian discovery quality and Inverse-Square node frequency for exploration.
- **Observability**: Implement structured `stdout` logging with Request ID and Type correlation.

## 2. Proposed Changes

### [ ] Configuration & Security
- Migrate `ADMIN_IDS` to `.env` (comma-separated list).
- Add `ACTION_CYCLE_MINUTES` (default: 60) to `.env`.

### [ ] Engine Refactor (Cycle-Based Logic)
- **Cycle Duration**: Replace hardcoded 1-hour with `ACTION_CYCLE_MINUTES`.
- **150-Cycle Window**: Update stats calculation to fetch the last 150 action log entries.
- **Cost**: `Food_Cost = max(1, 10 - Food_Efficiency_Level)`.
- **Output**: `int(20 * Stats_Eff * Quality_Eff * Building_Eff * Time_Ratio)` with base rate 20 for both Resources and XP.

### [ ] Exploring Rework
- **Discovery**: $P(\text{Found}) = 1 / (1 + n^2)$.
- **Quality**: Gaussian distribution based on `(PER+KNO)/2` with `Sigma=50` and `max(10, Quality)` clamp.
- **Stock**: `random(min=(PER+KNO)*10, max=(PER+KNO)*20)`.
- **Notification**: Automatic message to `announcement_channel` on any new discovery.

### [ ] UI/UX Enhancement (/idlevillage)
- **Layout**: Village Resources -> Buildings (Gaming names) -> Player Status (Bottom).
- **Refresh**: 5s cooldown button for UI-only settlement.
- **Timestamps**: Discord locale-aware timestamps for Last Activity and Next Check.

### [ ] Village Announcement System
- **Command**: `/idlevillage-announcement` (Admin only).
- **Format**: Mono-spaced code block with resource, building, and sorted villager status.
- **Throttling**: 60s cooldown for public message updates.

### [ ] Logging & Workflow
- **Logging**: Standardized format `[REQ_ID] [USER_ID/SYSTEM] {TYPE}: {MESSAGE}`.
- **Workflow**: Added developer testing guidance and cycle speed-up instructions.
