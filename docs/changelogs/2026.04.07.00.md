# Change Plan & Changelog: 2026.04.07.00

- Status: Draft

## 1. Objectives
- Completely simplify the core engine to a 1-hour resource lease model.
- Remove distance, travel symmetry, satiety, and weight systems.
- Establish a robust 1-hour pre-deduction and settlement loop for all actions.

## 2. Proposed Changes
- [ ] Database Schema Migration:
  - Simplify `players` table: Remove `satiety_deadline`, `current_weight`, `location_status`, `target_node_id`, `current_action_type`, `auto_restart`.
  - Add `target_id` (INTEGER) for nodes or buildings.
  - Remove `distance` and `max_capacity` from `resource_nodes`.
- [ ] Engine Refactor (`src/core/engine.py`):
  - Implement 1-hour cycle logic: pre-deduction at start, settlement at end.
  - Apply efficiency formulas: `(StatA + StatB) / 2 / 100`.
  - Food Efficiency Multiplier: `1 Food = 1.0 + (Level * 0.2) hours` of action duration.
  - Map actions to specific stats:
    - Building: KNO+END
    - Gathering Food: PER+KNO
    - Gathering Wood/Stone: STR+END
    - Exploring: AGI+PER
  - Implement building XP gain (50 XP base) and resource costs (10 Wood, 5 Stone, 1 Food).
  - Implement exploring logic (30m budget checks):
    - Miss (40% | 30m cost)
    - Lv1 Node (35% | 30m cost | 1000 stock)
    - Lv2 Node (15% | 60m cost | 2000 stock)
    - Lv3 Node (7% | 120m cost | 4000 stock)
    - Lv4 Node (3% | 240m cost | 8000 stock)
- [ ] UI Implementation (`src/cogs/actions.py`):
  - Update `/idlevillage` embed to reflect simplified village stocks and active 1-hour cycles.
  - Implement `/idlevillage-initial` with owner-only permission.
- [ ] Logic Refinement:
  - Implement automatic revert to `idle` if resources or node stock are insufficient.
  - Implement `missing` status for players inactive for 7 days.
