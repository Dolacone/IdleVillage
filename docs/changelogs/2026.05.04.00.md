# 2026.05.04.00 - Add efficiency line to personal info section

Status: Done

## Goal

The personal info section of the main embed (個人資訊) currently shows gear
levels, materials, current action, and AP. Players have no direct visibility
into their effective per-action output, which depends on three compounding
factors: stage clears, gear level, and facility level. Adding a 效率 line
exposes the computed output value and total bonus percentage per action type,
allowing players to understand how gear upgrades and building levels affect
their actual yield.

## Design

A new 效率 line is added as line 1 of the 個人資訊 section, immediately before
the existing 裝備 line.

New section template:

```
**個人資訊**
📊 效率：🌾 {n}(+{p}%) | 🔨 {n}(+{p}%) | ⚔️ {n}(+{p}%) | 🔬 {n}(+{p}%)
🏅 裝備：🌾 {n} | 🔨 {n} | ⚔️ {n} | 🔬 {n}
🎒 素材：🌾 {n} | 🔨 {n} | ⚔️ {n} | 🔬 {n}
🏃 行動：{emoji}{action_name}（下次結算：<t:{next_cycle}:R>）
⚡ AP：{ap} / 24
```

Where for each action type:

    n = floor(BASE_OUTPUT × (1 + bonus))
    p = floor(bonus × 100)
    bonus = stages_cleared × STAGE_BONUS_PER_CLEAR
          + gear_{action} × GEAR_BONUS_PER_LEVEL
          + facility_level(action) × FACILITY_BONUS_PER_LEVEL

Example with BASE_OUTPUT=20, 10 stages cleared, gathering gear Lv2,
gathering_field Lv3:

    bonus   = 10×0.01 + 2×0.03 + 3×0.01 = 0.19
    output  = floor(20 × 1.19) = 23
    display = 23(+19%)

All three inputs are already available in data passed to `build_main_embed`:
- stages_cleared  from stage_data
- gear levels     from player_row (gear_gathering, gear_building, gear_combat, gear_research)
- facility levels from buildings (gathering_field, workshop, hunting_ground, research_lab)
- env vars        via get_env_int / get_env_float (already imported)

No new DB queries or function signature changes are required.

Action-to-facility mapping follows engine/formula.md:
    gathering → gathering_field
    building  → workshop
    combat    → hunting_ground
    research  → research_lab

Emoji order is consistent with 裝備 and 素材 lines: 🌾 🔨 ⚔️ 🔬.

## Scope

### 1. Update build_main_embed in ui_renderer.py

Add a computation block in `build_main_embed` that derives the output value
and bonus percentage for each of the four action types using the data already
in scope, then formats the 效率 line and inserts it as line 1 of player_section
before the 裝備 line.

No changes to `build_main_components`, `build_village_embed`, or any other
function in the file.

### 2. Update docs/discord/ui-renderer.md

Add the 效率 line to the 個人狀態區 template. Reference engine/formula.md for
the formula definition.

## Out of Scope

- No change to formula.py or compute_output.
- No change to the village dashboard embed (build_village_embed).
- No change to DB schema or player_row structure.
- No change to tests unless an existing test asserts the exact embed string
  of the personal info section.
