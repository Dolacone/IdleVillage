# Change Plan & Changelog: 2026.04.08.02

- Status: Done

## 1. Objectives
- Implement notification system for resource node expiration.
- Enhance player lifecycle by notifying when a player transitions to idle.
- Improve UI readability by simplifying the active villagers section.
- Provide administrative tools for village resource management.
- Rebalance early-game progression with increased starting resources.
- Fix UI bug where idle players show "Manual refresh" instead of the next cycle countdown.

## 2. Proposed Changes

### [x] Resource Node Expiry Notifications
- Logic: When a resource node is depleted (out of stock) or expires (timeout) during a settlement cycle in `Engine.settle_player`, trigger a notification.
- Target: Send a message to the village's announcement channel.
- Content: Mention the node type, location (if applicable), and reason for expiry.

### [x] Player Idle Notification
- Logic: When a player's action completes and no new action is queued (transitioning to `idle` status), trigger a notification.
- Target: Tag the user in the village's announcement channel.
- Content: "You have finished your task and are now idle in the village."

### [x] UI Refinement: Active Villagers Section
- Change: In the `/idlevillage` dashboard and announcement embed, hide individual player stats (PER, KNO, etc.) from the active villagers list.
- Reason: Reduce line length and horizontal clutter in the Discord embed.
- Impact: Only show `[Status] Name (Time remaining/Progress)`.

### [x] Admin Command: `/idlevillage-admin`
- Description: [Owner Only] Manage village resources and nodes.
- Sub-commands:
  - `resource set <type> <amount>`: Set food, wood, or stone to a specific value.
  - `node remove <id>`: Manually remove a specific resource node.
- Implementation: Add to `src/cogs/general.py` with `is_admin` check.

### [x] Village Initialization Rebalance
- Change: Update `/idlevillage-initial` to set starting resources to 1000 Food, 1000 Wood, and 1000 Stone.
- File: `src/cogs/general.py`.

### [x] Global Rebalance: Base Output Adjustment
- Change: Increase the base output value for all actions (gathering, idle production, building XP) from 20 to 50.
- Reason: Accelerate early-game progression and increase the impact of player actions.
- Files: Update constants/formulas in `src/core/engine.py`, `docs/modules/resources.md`, and `docs/modules/buildings.md`.

### [x] Bug Fix: Idle State Next Check Display
- Bug: Idle players show "Next check: Manual refresh".
- Expected: Show the next cycle countdown even in idle state, as idle production still occurs.
- Implementation: Update `src/cogs/actions.py` to calculate next check time for idle players.
