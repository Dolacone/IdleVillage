# Change Plan & Changelog: 2026.04.10.00

- Status: In-Progress

## 1. Objectives
- Implement Monster System: Introduce hostile entities spawned during exploration that double village XP decay until defeated.
- Introduce Gold Resource: A high-tier resource obtainable exclusively via hunting monsters.
- Add Hunting Buff: A new village upgrade that increases damage dealt to monsters.
- Unified Interaction: Use the existing /idlevillage UI to handle both resource gathering and monster combat via an Interact action.
- Refactor Building Costs: Standardize all building cycles to consume dual resources (50+50), incorporating Gold into advanced upgrades.

## 2. Proposed Changes

### [ ] Monster Mechanics & Combat System
- Exploration Trigger (10%):
  - Rule: Only a complete action cycle (e.g., 10 mins) triggers discovery checks.
  - Spawn one of three monster types: Wild Boar (Food+Gold), Ancient Treant (Wood+Gold), Stone Golem (Stone+Gold).
  - Singleton Constraint: Only one active monster per village at any time.
- Monster Attributes:
  - HP (Health): BASE_OUTCOME (50) * random(15, 25) * Discoverer_Efficiency.
  - Quality (Defense): Capped at 175%.
  - Expiry: Monsters flee after 48 hours if not defeated.
- Combat Logic:
  - Linked Stats: Strength (STR) + Agility (AGI).
  - Damage Formula: 50 * Efficiency * (1.0 + Hunting_Level * 0.05) * ((200 - Quality) / 100) * Time_Ratio.
  - Multi-player: Settlement follows individual player timers. The first player to drop HP to 0 triggers rewards. Subsequent players in the same window become idle.
- Village Threat: While a monster is alive at the time of village settlement (tick), all building XP decay is doubled (x2).

### [ ] Resources & Rewards
- New Resource: Gold.
- Defeat Rewards:
  - Material (Food/Wood/Stone based on type) = HP_Max * (Quality / 100).
  - Gold = HP_Max * (Quality / 100).
  - Hunting XP = Fixed 1,000 XP (Village-owned).
- Initialization: New villages start with 0 Gold. (Obtainable only via Hunting).
- Storage Overflow Rule (v2026.04.10.00):
  - Logic: If a downgrade causes current stock to exceed the cap, existing resources are preserved, but no new resources are added.
  - Formula: `New_Stock = min(max(Capacity, Current_Stock), Current_Stock + Gain)`.

### [ ] Building Cost Refactor
- All building cycles consume 50 + 50 units:
  - Food Efficiency (ID 1): Food + Wood.
  - Storage Capacity (ID 2): Wood + Stone.
  - Resource Yield (ID 3): Wood + Stone.
  - Hunting (ID 4): Stone + Gold.

### [ ] Commands & UI Integration
- Existing /idlevillage Update:
  - The interaction menu will now show an Interact button.
  - If the target is a resource node, the action is labeled as Gathering.
  - If the target is a monster, the action is labeled as Attack.
- Status Rendering:
  - Combat status: [Attacking MonsterName] (HP: Current/Max).
  - Announcement board: New `Village Threat` section placed between `Last Update` and `Village Resources`.
