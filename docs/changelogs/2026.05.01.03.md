# 2026.05.01.03 - Discord: slash commands, routing, UI rendering

Status: Draft

## Goal

Replace the v1 Discord interaction surface with the v2 command and UI behavior from `docs/discord/command-handler.md` and `docs/discord/ui-renderer.md`.

## Implementation Gap

- `src/cogs/actions.py` still exposes v1 actions, token UI, and v1 player status.
- `src/cogs/general.py` still exposes v1 initialization and admin resource/node management.
- Current UI labels, controls, and embeds do not match v2.
- Single-guild enforcement through `DISCORD_GUILD_ID` is not implemented consistently.

## Scope

- Implement `/idlevillage`.
- Implement `/idlevillage-help`.
- Implement `/idlevillage-announcement`.
- Implement `/idlevillage-manage`.
- Enforce `DISCORD_GUILD_ID` for all commands and components.
- Render personal UI from v2 player, AP, materials, gear, and current action state.
- Render public dashboard from v2 stage, resources, buildings, and action counts.
- Catch up due complete cycles before opening or refreshing the UI.
- Apply refresh cooldown through `REFRESH_COOLDOWN_SECONDS`.
- Persist announcement channel and dashboard message references in `village_state`.

## Out of Scope

- No watcher loop implementation.
- No public notification dispatch beyond direct command responses.
- No v1 command compatibility.

## Files

- `src/cogs/actions.py`
- `src/cogs/general.py`
- possible new Discord UI modules under `src/cogs/` or `src/discord/`
- focused tests under `tests/`

## Verification

- `/idlevillage` opens the v2 personal UI ephemerally.
- Opening and refreshing the UI catches up due full cycles before rendering.
- Action change routes through partial settlement first.
- Burst button spends AP and runs three complete settlements.
- Gear upgrade UI calls the gear manager and displays current state.
- Admin resource controls use env-configured small and large deltas.
- Commands outside `DISCORD_GUILD_ID` are rejected.

## Dependencies

Requires `2026.05.01.00`, `2026.05.01.01`, and `2026.05.01.02`.
